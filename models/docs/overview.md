# Regulatory payments analytics

This project transforms synthetic customer and payment data into **documented, tested reporting marts**.

## What it demonstrates

- Modular SQL transformations using `source()` and `ref()`
- Standardisation of country codes and currency routes
- Duplicate handling and referential-integrity validation
- A trustworthy payment fact table plus regulatory, operational and exception marts
- Automated data tests and an interactive dbt lineage graph

All data in this repository is fictional and contains no personal or confidential information.
